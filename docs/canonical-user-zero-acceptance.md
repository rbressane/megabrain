# Canonical User-Zero Acceptance

All acceptance fixtures are synthetic. No private Brain or source vault is used.

| Normal question | Required result |
|---|---|
| What are all current prices and when should each menu be used? | Complete bounded collection; unrelated core and unauthorized sensitive records absent. |
| How do I recover an agent after a failure? | Recovery runbook metadata resolves and its stable URI opens. |
| What is the current fleet update policy and runbook? | Current policy/runbook resource is returned with verification metadata. |
| What are the owner's active ventures and next actions? | Relevant current memory is returned within the budget. |
| What canonical knowledge exists for a named project? | Project resources list without unrelated documents. |
| What is the relationship between legacy sources and MegaBrain? | Canonical authority/migration resource is returned. |
| Open the canonical runbook for a named project. | `resource-read` returns the current immutable revision as untrusted data. |
| Show the archived source artifact supporting this decision. | Archive resource or attachment manifest resolves with provenance and fingerprint. |

`tests/test_megabrain.py` covers the eight-item pricing collection and core/sensitivity boundaries. `tests/test_canonical.py` covers the remaining normal-language questions, source evidence, policy, migration, rollback, and security failures.
